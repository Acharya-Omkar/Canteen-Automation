                  Magnus Eriksson • a year ago

                  Just a heads up, this code is pretty insecure.

                  1. _Never_ use md5() for password hashing. md5() is a not only super fast (which makes it easy to brute force) but it's also broken (there have been hash collisions). The lack of salt here also makes your users passwords pretty easy to "de-hash" using one of the thousands of rainbow tables you can find online.
                  Use PHP's password_hash() and password_verify() instead. They are both well documented in the manual.

                  2. Don't escape passwords since that changes the password. Since you're saving the password hash and not the actual password, there's no need to escape it.

                  3. Use parameterized Prepared Statements instead of manually escaping the user data and concatenating the sql query. Prepared statements is currently the safest way. Even mysqli_real_escape_string() has security issues in certain cases (sure, those are edge cases, but still). Since Prepared statements actually solves the whole escaping issue by sending the query with the place holders and the actual data separately to the database, injections isn't an issue.

                                  Magnus Eriksson Denise Silva • 6 months ago

                                  How are you verifying the passwords? When you hash a password with password_hash(), it will always generate a new unique hash. So when verifying the user, the flow need to be:

                                  1. Fetch the user from the database based on the username only.
                                  2. Use password_verify() to compare the hash you just got from the database (for that username) with the password the user just entered.

                                  If password_verify() returns true, it's the correct credentials.

                                  So basically, the difference is that you can't compare the hash in the database query directly like you do with a md5 hashes since you never will get the same hash, even for the same password.

                                  I recommend reading the manual about password_hash() and password_verify() properly to really understand how those functions work. Don't take security lightly.

                                                                                                                                            Hello friend, my apologies for not replying to your comment. Inside our server.php file, there are two places where the user logs in. One of the places is when we register the user and they are automatically logged in, and the other is when the user enters their details in the login form and clicks the login button.

                                                                                                                                            In both of these places in our server.php file, we are selecting all info from the users table (SELECT * FROM users...) the star (*) means all database fields, and we are only storing the username in the session variable when we do this:
                                                                                                                                            $_SESSION['username'] = $username;

                                                                                                                                            If you want to store another attribute (say email for example) in the session, you just add another line like this
                                                                                                                                            $_SESSION['email'] = $email;
                                                                                                                                            and so on. That is just it.

                                                                                                                                            $query = "INSERT INTO users (username, email, password)
